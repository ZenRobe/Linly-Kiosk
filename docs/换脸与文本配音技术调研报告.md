# 换脸与文本配音技术调研报告

---

## 一、换脸方案调研

### 1.1 四种方案对比总览

| 维度 | Deep-Live-Cam | FaceFusion | Qwen+BFS LoRA 逐帧 | FaceFusion + Qwen 精修 |
|------|:---:|:---:|:---:|:---:|
| **定位** | 实时摄像头换脸 | 视频精修 + 实时 | 极致画质逐帧换脸 | 粗换 + 精修两阶段 |
| **实时性** | 30 FPS | 25 FPS | 非实时（秒/帧） | 实时粗换 + 离线精修 |
| **画质** | 中等 | 高 | **最高**（原始分辨率） | **最高**（精修后） |
| **最低显存** | 4GB | 8GB | 24GB+ | 24GB+ |
| **许可证** | AGPL-3.0 | AGPL-3.0 | Apache 2.0 | 混合 |
| **适用场景** | 轻量实时交互 | 高质量视频输出 | 工业化离线精修 | 实时+离线混合 |

---

### 1.2 Deep-Live-Cam（轻量实时型）

> 地址：https://github.com/hacksider/Deep-Live-Cam

**定位**：GitHub 最热门的实时换脸项目（72K Star），仅需单张照片即开即用。

- **优**：显存门槛低（最低 4GB），RTX 3060 可达 1080p/30FPS
- **劣**：画质中等，侧脸/遮挡易穿帮，后处理仅 GFPGAN

---

### 1.3 FaceFusion（高质量全能型）

> 地址：https://github.com/facefusion/facefusion

**定位**：下一代替换脸与增强平台（35K Star），支持离线精修和实时摄像头。

- **优**：画质业界标杆，内置多款模型可选（inswapper / GhostFaceNet / HyperSwap）；后处理链完整（GFPGAN / CodeFormer / 泊松融合 / 颜色校正）；内置唇形同步、表情保留器
- **劣**：最低 8GB 显存；v3.5 webcam 模式 1080p/25FPS

---

### 1.4 Qwen-Image-Edit-2511 + BFS LoRA 逐帧换脸（画质旗舰型）

**底层模型**：Qwen-Image-Edit-2511（阿里通义千问，2025 年 12 月开源）
**LoRA**：BFS Best Face Swap（社区 @Alissonerdx 开发，专为 Qwen 定制的换脸 LoRA）

> Qwen 模型地址：https://huggingface.co/Qwen/Qwen-Image-Edit-2511
> BFS LoRA 地址：https://huggingface.co/Alissonerdx/BFS-Best-Face-Swap

**工作原理**：
```
视频 → 拆帧(FFmpeg) → 逐帧: Qwen-Image-Edit + BFS LoRA 换脸 → 合帧 → 输出视频
```

**核心优势**：
- **画质上限最高**：基于 200B 参数扩散模型，输出可达 2K/4K 原始分辨率，皮肤纹理、光影过渡、五官细节远超传统换脸方案
- **身份一致性极强**：2511 版本的核心突破就是"人物一致性"，修图过程中不易发生面部漂移

**关键短板**：
- **非实时**：单帧推理需数秒（RTX 4090 上约 2-5 秒/帧，20-25 采样步数），1 分钟 30FPS 视频需 1800 帧 × 3 秒 ≈ 1.5 小时
- **显存需求极高**：模型约 40.9 GB（FP16），需 **24GB+ 显存**（RTX 4090 / 5090）
- **帧间一致性挑战**：独立处理每帧易产生闪烁/抖动，需额外引入时序平滑模块（如光流插值）
- **加速空间有限**：LightX2V 闪电 LoRA 可将步数降至 4 步，但画质有所折损

---

### 1.5 FaceFusion（HyperSwap）+ Qwen 2511 精修（混合型）

**工作流程**：
```
视频 → FaceFusion(HyperSwap) 粗换 → 选取关键帧 → Qwen 2511 精修 → 帧间插值 → 输出
```

**设计思路**：
- **第一阶段**：FaceFusion + HyperSwap 完成全视频快速换脸（实时或准实时）
- **第二阶段**：Qwen 2511 仅对关键帧（如正脸特写、大角度帧）进行精修，提升细节质量
- 精修后的关键帧与其余粗换帧通过插值/融合拼接

**相比纯 Qwen 逐帧的优势**：
- 大幅减少 Qwen 推理次数，耗时降至纯逐帧方案的 5%-20%
- 保留 FaceFusion 的实时性基线，Qwen 仅在需要时介入
- 帧间一致性由 FaceFusion 保障，Qwen 只做增强不做替换

**权衡**：
- 需要设计关键帧选取策略（如人脸角度检测、清晰度评估）
- 两套系统的集成复杂度高于单一方案
- 仍需 24GB+ 显存部署 Qwen 2511

---

### 1.6 换脸模型对比

| 模型 | 分辨率 | 画质级别 | 推理速度 | 来源 |
|------|:---:|------|------|------|
| **inswapper** | 128×128 | 基准 | 快（8-12ms） | InsightFace |
| **HyperSwap** | 256×256 | 提升约 2 倍 | 较慢（+50%） | FaceFusion Labs |
| **Qwen+BFS LoRA** | 原始分辨率 | **最高**（细节/光影全面碾压） | 秒级（非实时） | 阿里通义 + 社区 |

---

### 1.7 画质增强（各方案共用）

| 增强模块 | 方案 | 效果 | 性能代价 |
|------|------|------|------|
| **GFPGAN** | 前三种方案 | 修复边缘伪影、纹理不自然 | 帧率减半 |
| **Qwen 2511 精修** | 混合方案 | 全局画质提升，细节远超 GFPGAN | 秒级延迟 |

---

### 1.8 与本项目适配评估

| 维度 | 评估 |
|------|------|
| **技术契合** | 均为 Python 技术栈，DLC/FaceFusion 可封 Gradio 模块；Qwen 需额外推理服务 |
| **离线部署** | 全部支持本地部署，无网络依赖 |
| **实时性** | Kiosk 场景（用户触发→换脸人脸展示）：DLC 30FPS / FaceFusion 25FPS 满足；Qwen 逐帧不满足实时 |
| **画质要求** | 竖屏 2160×3840 对画质要求高，HyperSwap 256px 为底线，Qwen 精修为上限 |
| **硬件成本** | DLC 最低 / FaceFusion 中等 / Qwen 方案需 RTX 4090+（约 1.5 万+） |

---

### 1.9 推荐方案

| 硬件配置 | 推荐方案 | 理由 |
|------|------|------|
| **RTX 3060（6-12GB）** | FaceFusion + HyperSwap + GFPGAN | 最低成本跑通高质量换脸 |
| **RTX 4090（24GB）** | FaceFusion + HyperSwap（实时） + Qwen 2511（精修关键帧） | 混合方案，实时与画质兼顾 |
| **纯离线精修场景** | Qwen-Image-Edit-2511 + BFS LoRA | 画质天花板，不计时间成本 |

> **对本项目的建议**：Kiosk 为交互式展示场景，需实时响应。若预算允许配备 RTX 4090，首选 **方案二（FaceFusion + Qwen 精修）**——实时粗换保证流畅体验，Qwen 精修保证展示画质。

## 二、文本配音（TTS）技术调研

6 款在线网页 TTS 工具，免费/轻量方向，适合快速试听和配音制作。

---

### 2.1 工具对比总览

| 工具 | 免费额度 | 音色数 | 需注册 | 下载 | 特色 |
|------|------|:---:|:---:|:---:|------|
| **TTSMaker** | 每周 3 万字符 | 300+ | 否 | MP3/WAV/OGG | 商用版权、API 接口 |
| **Text2Voice** | 无限 | 20+ 中文 | 否 | MP3 | 微软 Edge TTS 技术，语速/音调/风格可调 |
| **AI-speaker** | 免费 | 基础 | 否 | MP3 | 极简，即开即用 |
| **剪映** | 大部分免费 | 数十种 | 需装客户端 | 直接生成到音轨 | 与视频剪辑无缝衔接、AI 克隆音色 |

> \*NoteGPT 实际免费额度极低，注册后易被引导付费，Trustpilot 评分仅 2.5/5，**慎用**。

---

### 2.2 各工具详评

#### TTSMaker（推荐）

- **地址**：ttsmaker.cn
- **定位**：国内最成熟的免费 TTS 工具之一，300+ 音色覆盖 50+ 语言，含方言、二次元、情感化等特色音色
- **核心优势**：
  - 每周 3 万字符免费额度（约 150 分钟音频），部分热门音色无限免费用
  - **商用版权明确**：免费生成的音频 100% 版权归用户，可商用
  - 提供 **API 接口**，可集成到自有系统（付费）
  - 支持语速（0.5x-2x）、音量、音调精细调节，智能停顿插入
  - 多格式导出，生成文件 1 小时内自动删除保隐私
- **短板**：免费版单次上限约 1 万字符；在线工具需网络

#### Text2Voice（Edge TTS 技术）

- **底层**：微软 Azure 神经网络语音合成（Edge TTS），免费无限制
- **核心优势**：
  - 音质高（神经网络合成，接近真人），生成速度快
  - 21 种中文语音（晓晓/云希/云健等），11 种风格（新闻腔/客服/温柔/活泼等）
  - 完全免费、免注册、调用次数无硬限制
  - 有成熟的 Python 库 `edge-tts`（`pip install edge-tts`），**可编程调用，便于集成到本系统**
- **短板**：在线服务，需网络；商用需注意微软服务条款

#### 剪映

- **定位**：字节跳动旗下视频剪辑工具，内置文本朗读
- **核心优势**：
  - 多数音色免费，支持 AI 克隆音色（录制约 100 句即生成专属声音）
  - 语音直接生成到音频轨道，配音与剪辑流程无缝衔接
  - 情感化发音、语速/语调可调
- **短板**：需安装客户端（非纯网页）；部分高级音色需 SVIP；无 API 接口，**不适合程序化集成**



#### AI-speaker（轻量备选）

- 极简在线工具，无需登录，基础 TTS 需求足够
- 音色选择有限，适合临时快速生成

---

### 2.3 与本项目适配评估

| 维度 | 评估 |
|------|------|
| **集成方式** | Text2Voice 的 `edge-tts` Python 库可直接在 `app.py` 中编程调用，实时合成音频，与 Gradio 架构天然兼容；TTSMaker API 次之（需付费） |
| **离线部署** | 上述工具均为在线服务，需网络连接。若 Kiosk 要求纯离线，需换用本地 TTS 引擎（如 Piper TTS、Coqui TTS 等） |
| **音质要求** | Kiosk 为公共服务展示，Edge TTS 神经网络音质已足够；若需顶级音质可考虑商业 API（如 ElevenLabs），但存在成本 |
| **成本** | `edge-tts` 完全免费、无调用限制；TTSMaker 免费额度可满足常规配音需求 |
| **合规** | Edge TTS 商用需确认微软条款；TTSMaker 明确免费音频可商用 |
| **维护** | 在线服务无需本地维护模型，但依赖第三方服务稳定性 |

---

### 2.4 推荐方案

> Kiosk 配音场景为「预设问题朗读 + 答案播报」，文本固定、非实时生成。推荐统一配音流程：**前期用 TTSMaker（选音色、调节奏）+ 导出 MP3 → 存入本地**，避免在线依赖。若需程序化实时合成（如动态答案），选用 **`edge-tts` Python 库**集成到 `app.py`。
